Fotbalové Kartičky

A personalized football trading card platform for Czech youth players, parents, coaches, and clubs — combining real match stats, AI-powered styling, gamification, and physical print export.

Next.js 16 TypeScript 5 Firebase Konva.js OpenAI GPT-4o-mini Tailwind v4 Czech & English PWA
About

What is this app?

Fotbalové Kartičky (Football Cards Forge) lets young Czech football players create collector cards from real match data, share them socially, and order professional prints.

Target audience

Children aged 6–16, their parents and coaches, and Czech football clubs managing youth teams. Content is Czech-first with full English translation.

Core concept

Players earn real collector cards from actual match statistics. Better performance unlocks rarer card types (Common → Rare → Epic → Legendary) with special visual effects.

Gamification

Every action earns XP. Players level up (Začátečník → Legenda), unlock new card styles, and earn achievement badges — keeping young players engaged all season.

Monetization

Freemium model: Free (3 cards/month), Premium (149 CZK/mo), Team (399 CZK/mo), Club (custom). Tier gating via Firebase Remote Config.

Features

Main capabilities

Sixteen implementation phases covering the full product surface.

👤

Player Management

Create player profiles with photos, positions, and biographical data. Photo upload to Firebase Storage with live preview.

🏆

Team & Match Tracking

Organize players into teams, record match results, and enter per-player stats (goals, assists, rating, cards, minutes).

🃏

Card Generation

Multi-step wizard: select player → pick template → configure stats → generate. Rarity auto-calculated from performance.

🎨

Card Editor (Konva)

Layered canvas editor (420×588px) with background, photo, stats, text, and visual effects. Exports at 3× resolution.

✨

Visual Effects

Five Konva filter-based effects: Gold Foil, Hologram, Neon, LED, and Fire — applied per card rarity or manually.

🤖

AI Studio

Background removal and artistic style transfer via Replicate API. OpenAI GPT-4o-mini for card text generation. All via Cloud Functions — API keys never reach the client.

📦

Collection & Albums

Full collection view with filters (rarity, player, season), season album grouping, hologram CSS effects, and 3D card flip on hover.

🏅

Achievements & XP

Event-driven achievement engine. XP earned for every action. Level progression from Začátečník to Legenda. Unlockable card styles per level.

📤

Social Sharing

Native Web Share API with fallback to clipboard. Platform-specific links for Facebook, WhatsApp, Messenger, TikTok. QR code generator and download.

🖨️

Print Center

jsPDF export in four layouts: A4 (9-up), A4 (4-up large), Sticker sheet with cut marks, and A4 landscape Album. Tier-gated formats.

💳

Subscription Engine

Four tiers (Free/Premium/Team/Club) with limits on cards/month, AI credits, print formats, players, and teams. All gating in one central module.

🔔

PWA & Push Notifications

Installable PWA with offline caching via Service Worker. Firebase Cloud Messaging for background push notifications with permission request flow.

Architecture

Key decisions

Constraints and patterns that shape all development decisions.

  • Client-side auth only — no Admin SDK, no session cookies AuthProvider listens to onAuthStateChanged → writes to Zustand. Route guards live in AppShell (client component), not Next.js middleware. Fix for race conditions: setUser() before router.push() on registration.
  • Single Firestore access point All reads and writes go through src/lib/firebaseServices.ts. Components and stores never import db directly. Every document has createdAt, updatedAt (Timestamp) and userId/ownerId.
  • API keys server-side only OPENAI_API_KEY and REPLICATE_API_KEY are Cloud Functions environment variables only — never in .env.local or any client bundle. All AI calls go through Cloud Function HTTP endpoints.
  • One real-time listener — AI status only While a card's aiStatus is "pending" or "processing", the card detail page opens an onSnapshot on cards/{cardId}. The listener is closed when aiStatus becomes "done" or "error".
  • Konva Stage initialized once The card editor's Konva Stage is created in a useEffect via useRef and never recreated on re-render. Card dimensions are fixed at 420×588px (2.5:3.5 ratio). Export at pixelRatio 3 for print quality.
  • Subscription gating centralized All tier limit logic lives in src/lib/subscriptionEngine.ts. Components call gate functions (canGenerateCard, canUseAI, canPrint) and show UpgradeDialog on failure. Never hardcode tier limits in components.
  • i18n: proxy.ts not middleware.ts next-intl requires the middleware file to be named src/proxy.ts. All app routes live under src/app/[locale]/. User-facing strings always via useTranslations() — never hardcoded.
