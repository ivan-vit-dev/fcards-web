Architecture Overview

Guiding constraints and decisions — read this before touching the code.

Auth model

Client-side only. AuthProvider wraps onAuthStateChanged → useAuthStore. Route guards in AppShell (client component). No middleware auth. No Admin SDK. No session cookies.

Firestore access

All reads/writes via src/lib/firebaseServices.ts only. Never import db in components/hooks/stores directly. Every document has createdAt, updatedAt (Timestamps) and userId/ownerId.

AI calls

All OpenAI and Replicate calls made from Firebase Cloud Functions only. Client calls HTTP endpoint → gets jobId → opens onSnapshot on card doc watching aiStatus. API keys never in .env.local or client bundle.

Real-time listeners

Only one: card detail page opens onSnapshot on cards/{cardId} while aiStatus === "pending" | "processing". Closed when status becomes "done" | "error".

Canvas editor

Konva Stage initialized once via useRef in useEffect. Never recreated on re-render. Dimensions: 420×588 px (2.5:3.5 card ratio). Export: stage.toBlob({ pixelRatio: 3 }).

Subscription gating

All tier limits in src/lib/subscriptionEngine.ts. Gate with canGenerateCard(tier, count), canUseAI(), canPrint(). Show <UpgradeDialog> on failure. Never hardcode limits in components.

XP & Achievement Engine

src/lib/xpEngine.ts · src/lib/achievementEngine.ts

XP Engine

addXP(uid, amount)          → void
getLevelFromXP(xp)          → number
getUnlockables(level)       → Unlockable[]

Level progression: XP thresholds increase exponentially. Each level unlocks card styles, effects, or features. getLevelTier(level) in utils.ts returns tier string (Začátečník → Legenda).

Achievement Engine

checkAchievements(user, event) → Achievement[]

Pure function — no side effects. Called after every significant action (card created, match recorded, etc.). Returns list of newly earned achievements. Caller writes to userAchievements via Cloud Function (admin SDK bypasses Firestore rules).

Print Engine

src/lib/printEngine.ts

exportCardsToPDF(
  cards: Card[],
  format: PrintFormat,
  onProgress: (pct: number) => void
): Promise<void>

Fetches each card image via fetchAsDataUrl(card.imageUrl) (fetch → FileReader → base64). Renders cards on jsPDF pages using computed LAYOUTS[format] dimensions. For sticker format, calls drawCutMark(doc, x, y) at each card corner. Increments user.printCount on completion.

Security Rules

Firestore (firestore.rules)

Key patterns:

users/{uid} Read: owner only. Write: owner only. Role cannot be self-upgraded.
players/{playerId} Read: owner or public (isPublic==true). Write: owner only.
cards/{cardId} Owner: full access. Public cards: anyone signed-in may increment shareCount by exactly 1. Rule: diff.affectedKeys().hasOnly(['shareCount','updatedAt']) && shareCount == old.shareCount + 1.
userAchievements/{uid}/{achievementId} allow write: if false — written only by Cloud Functions admin SDK, which bypasses rules.

Storage (storage.rules)

users/{uid}/photos/ Owner read/write. Max 10 MB. Content-type must match image/.*.
cards/{cardId}/ Owner read/write. Max 20 MB. Image only.
templates/{templateId}/ Public read. allow write: if false — written by admin scripts only.

Patterns & Gotchas

Button navigation — never use asChild

shadcn Button does NOT support asChild

Always use useRouter().push() with onClick. Never wrap <Link> inside <Button asChild>.

// WRONG
<Button asChild><Link href="/foo">Go</Link></Button>

// CORRECT
const router = useRouter();
<Button onClick={() => router.push("/foo")}>Go</Button>

No setState in useEffect

react-hooks/set-state-in-effect ESLint rule

Do not call setState inside a useEffect body — even indirectly via a function that internally sets state. Use lazy useState initializers for reads-at-mount. Keep all setState calls in event handlers or callbacks.

Opacity with CSS variable colors

Tailwind opacity modifiers don't work with CSS vars

bg-primary/50 does NOT work with oklch() CSS variable tokens.
Use: color-mix(in oklch, var(--token) 50%, transparent) in raw CSS instead.

Race condition on registration

Always setUser before router.push

After registration or login, call setUser(userDoc) BEFORE router.push("/dashboard"). AppShell checks the store synchronously — if user isn't in the store yet, it will redirect back to login.

lucide-react has no brand icons

Facebook, TikTok, WhatsApp — use inline SVGs

lucide-react removed brand icons. ShareButtons.tsx contains inline FacebookIcon and TikTokIcon SVG components. Do the same for any other brand icon needed.

next-intl middleware filename

src/proxy.ts — not src/middleware.ts

next-intl requires the middleware export from src/proxy.ts. If renamed to middleware.ts, locale routing breaks silently.

Firestore writes — never import db directly

All reads/writes via firebaseServices.ts

Components, stores, and hooks must never import db from firebase.ts. Only src/lib/firebaseServices.ts and src/hooks/use-ai-generation.ts (for the one real-time listener) are exceptions.
